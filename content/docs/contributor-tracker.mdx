# Contributor Tracker

The Contributor Tracker is a feature that automatically displays contributors for each documentation page. It fetches contributor data from the GitHub API and displays it in a user-friendly format.

<ContributorTracker docPath="contributor-tracker.mdx" />

## How It Works

The Contributor Tracker works by:

1. Fetching commit history for a specific documentation file from the GitHub API
2. Extracting unique contributors from the commit history
3. Displaying contributor information including:
   - Avatar
   - Username
   - Contribution count
   - Last commit date

## Using the Contributor Tracker

To add a contributor tracker to any documentation page, simply include the following component in your MDX file:

```mdx
<ContributorTracker docPath="path/to/your/document.mdx" />
```

If you omit the `docPath` prop, it will automatically use the current document's path.

## API Integration

The Contributor Tracker uses the following API endpoint to fetch contributor data:

```
GET /api/contributors?docPath=path/to/document.mdx
```

This endpoint returns a JSON array of contributors:

```json
[
  {
    "id": 12345,
    "login": "username",
    "avatar_url": "https://avatars.githubusercontent.com/u/12345?v=4",
    "html_url": "https://github.com/username",
    "type": "User",
    "contributions": 5,
    "last_commit_date": "2025-10-27T10:30:00Z"
  }
]
```

## Customization

The Contributor Tracker component can be customized by passing additional props:

- `className` - Add custom CSS classes to the container
- `docPath` - Specify the document path to fetch contributors for

## Benefits

- **Automatic Updates**: Contributor information updates automatically as new commits are made
- **Performance**: Data is cached to minimize API calls
- **User Engagement**: Encourages contributions by giving credit where it's due
- **Transparency**: Shows the community effort behind each documentation page
